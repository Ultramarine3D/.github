## About the Project

Ultramarine3D is an open-source FFF (Fused Filament Fabrication) 3D printer built on a CoreXY motion system. The project exists to make high-quality 3D printing more accessible by removing the black boxes — every part, every line of firmware, and every design decision is documented and open for the community to inspect, modify, and improve.

We believe 3D printing hardware should be:

- **Transparent** — every component, schematic, and CAD file is publicly available
- **Approachable** — firmware and electronics are designed to be understood, not just used
- **Honest** — documentation reflects what actually works, including the trade-offs

Whether you're a maker, an engineer, or someone who just wants to understand what's inside their printer, Ultramarine3D is built to be opened up.

## Key Features

- **CoreXY motion system** for fast, precise movement with a stationary build plate on the Z-axis
- **Fully open hardware** — mechanical CAD, BOM, and assembly guides published openly
- **Approachable firmware** with clear configuration and documented behavior
- **Modular design** that supports community modifications and upstream contributions
- **Honest documentation** — known issues and limitations are published alongside the wins

## Getting Started

Once documentation is available, the typical workflow will be:

1. Review the **Bill of Materials** and source the required parts
2. Follow the **Assembly Guide** to build the frame, motion system, and electronics
3. Flash the recommended **firmware** using the provided configuration
4. Run through the **Calibration** procedure for your first print

If you'd like to be involved before the public docs land, see [Contributing](#contributing).

## Contributing

Ultramarine3D is community-driven, and contributions of all kinds are welcome:

- **Hardware** — improvements to the CAD, frame, or motion system
- **Firmware** — bug fixes, new features, configuration profiles
- **Documentation** — assembly notes, calibration tips, troubleshooting guides
- **Testing** — building the printer and reporting back on what worked (and what didn't)

If you'd like to contribute, please open an issue to discuss your idea before submitting major changes. A formal `CONTRIBUTING.md` will be added as the project matures.
